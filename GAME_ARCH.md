# 數論密室 · 遊戲架構 v2（教學導向重構）

> 此文是**機制／教學架構**藍圖，與 `LEVEL_BLUEPRINT.md`（章節內容）互補。
> v1 的問題：遊戲把「選運算／算數值／為什麼安全」三件事都替學生做掉，認知層級停在 guided Apply。
> v2 目標：把這三件事**還給學生**，用「步數」取代「能量」，讓試錯免費、讓難問題的「難」可被體感。
> 決定定案 2026-06-17。

---

## 0. 設計哲學

**一句話**：試錯免費，精煉計分；引擎當裁判不當保母；難問題用步數讓玩家親手摸到。

三條地基原則：

1. **零懲罰探索**：沒有能量上限、可無限出牌、自由 undo/reset。過關只看「有沒有達標」，不看「省不省」。
2. **步數即量尺**：達標後回報 `{steps, par, best}`。步數對破解關是「演算法難度的儀器」——小參數十幾步可行、大參數爆百步 → 玩家自己發現要更聰明的解法。
3. **謂詞即勝利**：勝利＝一個 `Predicate(board)` 成立，而非「湊出某個指定數值」。讓破解、偽造、回報無解都能用同一框架表達。

## 1. 鎖定的決定（DECISIONS）

| # | 決定 | 取捨 |
|---|---|---|
| D1 | **零能量 + 記步數 + par** | par 直接取每關標準解 `path.length`，零額外資料。過關＝解開；步數只是精煉度 flair，不是門檻。 |
| D2 | **引擎當裁判**：誠實算 BigInt，但不預檢「這步在不在標準解上」 | 允許算垃圾、走死路。試錯要被允許且便宜。 |
| D3 | **勝利＝謂詞框架** | `equals / tokensEqual / inverts / forges` 四型，涵蓋破解與偽造，引擎不為每關特判。 |
| D4 | **手牌是超集，路徑不寫死** | 發的卡多於所需、含誘餌；「選哪個運算」本身就是學習。 |
| D5 | **去公式化 + 漸進提示** | story 只給情境與目標、不給公式；卡死可逐級揭示提示並記 `hintsUsed`。 |
| D6 | **破解關兩段式** | 先親手窮舉（小參數，體感暴力可行）→ 過關解鎖萬能卡當「獎勵工具」（大參數用，體感演算法 > 暴力）。 |
| D7 | **王關加「為什麼」閘** | 達標外再答一個概念挑戰（可內建成謂詞，如 e 與 φ 不互質時正解＝回報「無解」）。 |

## 2. 資料模型

```js
// 盤面棋子：值可為 BigInt 或橢圓曲線點（沿用 v1 的 E3 點型別）
Token = { id, value: BigInt | Point, label, kind? }

// 一步操作（undo / 步數 / PDF 轉錄 三用，相容現有 transcript）
Move  = { card, inputs: Token[], output: Token }

// 勝利謂詞（取代 v1 的 inst.target 直接比對）
Predicate =
  | { type:'equals',      value }                 // 盤面出現某值
  | { type:'tokensEqual', a, b }                  // 兩 token 同值（DH/驗章）
  | { type:'inverts',     f, y }                  // 盤面有 x 使 f(x)=y（破解，靠搜）
  | { type:'forges',      verify }                // 盤面有 (m,sig) 通過 verify（偽造）
  | { type:'impossible' }                         // 正解＝回報無解（why 閘）

// 關卡定義
Level = {
  id, title, story,        // story 去公式化：只給情境＋目標
  seeds: Token[],          // 初始盤面
  hand:  CardId[],         // 超集，含誘餌卡
  goal:  Predicate,
  par:   number,           // = 標準解步數，供計分
  hints: [string,string,string],   // Lv1 圈卡 → Lv2 公式骨架 → Lv3 完整解
  search?: SearchSpec      // 破解關才有，見 §5
}

// 每關成績（localStorage）
Score = { solved:boolean, steps:number, par:number, hintsUsed:0|1|2|3, best:number }
```

## 3. 狀態機與核心迴圈

```
GameState
 ├ progress: Set<chapterId>              // 已解鎖章
 ├ tools:    Set<cardId>                 // 兩段式解鎖的「獎勵工具卡」（factor/dlog/ecdlog…）
 ├ scores:   { [levelId]: Score }
 └ run:
     ├ board:   Token[]
     ├ history: Move[]
     ├ hand:    CardId[]
     └ goal:    Predicate

迴圈：
  選卡 → 填槽 → COMMIT
       → out = engine.apply(card, inputs)     // 誠實算，不檢查對錯
       → board.push(out); history.push(move)
       → if goal(board): settle()             // 結算 steps vs par
  undo  : history.pop(); board.pop()
  reset : board ← seeds; history ← []          // 步數歸零，鼓勵重試
  hint  : 揭示 hints[k]；score.hintsUsed = max(used, k+1)

settle():
  score = { solved:true, steps:history.length, par, hintsUsed, best:min(best,steps) }
  顯示：✔ 過關 · 你花了 {steps} 步（最佳解 {par} 步）{steps<=par?'　★完美':''}
  解鎖下一關；若此關標 unlockTool，tools.add(toolCardId)
```

**關鍵**：`history` 單一結構同時供 undo／步數／PDF 轉錄，與現有匯出機制相容，不必重寫。

## 4. 引擎改動（相對 v1）

| 改動 | v1 現況 | v2 |
|---|---|---|
| 勝利判定 | `valEq(token, inst.target)` | `evalPredicate(goal, board)` ——四型分派 |
| 資源 | energy 扣到 0 失敗 | 移除 energy 欄位與 UI；改記 `history.length` |
| 步檢 | 每步保證在可解路徑 | 移除；任何合法輸入都接受，死路允許 |
| 手牌 | path 寫死所需卡 | hand 為超集，含誘餌；path 僅用來算 par 與生提示 |
| story | 內含公式（`m=(c−k) mod 26`） | 只留情境；公式移進 `hints[1]` |

`evalPredicate` 草圖：
```js
function evalPredicate(goal, board){
  switch(goal.type){
    case 'equals':      return board.some(t => valEq(t.value, goal.value))
    case 'tokensEqual': return board.some(a=>board.some(b=>a!==b && valEq(a.value,b.value) && matches(a,goal.a)&&matches(b,goal.b)))
    case 'inverts':     return board.some(t => valEq(goal.f(t.value), goal.y))
    case 'forges':      return pairs(board).some(([m,s]) => goal.verify(m.value, s.value))
    case 'impossible':  return /* 由玩家按「回報無解」鈕觸發，引擎驗證真的無解 */ false
  }
}
```

## 5. 破解關兩段式機制（D6 核心）

公鑰密碼的靈魂＝陷門單向。用**同一把步數量尺**讓玩家依序體感三種難度 regime：

```
階段 A · 親手窮舉（小參數）        階段 B · 獎勵工具（中→大參數）
─────────────────────────        ──────────────────────────────
只給正向卡（modpow）+ 搜索迴圈     過關後 tools.add('dlog')
玩家逐一試 x：每試 +1 步           中參數：手搜 O(p) 爆步 → 工具卡 O(√p) 一鍵
~10 步搜出 → 「暴力可行」           大參數：連工具卡都吃力 → 「為何要大參數＝為何安全」
```

`SearchSpec`（階段 A 的搜索迴圈 UI）：
```js
SearchSpec = {
  forward: (x)=>BigInt,   // 玩家能算的正向函數，如 x => g**x % p
  target:  BigInt,        // 要反推到的 h
  range:   [lo,hi]        // 候選範圍；UI 給「試這個 x」按鈕，命中即 inverts 成立
}
```
- 階段 A 關**不發** `dlog`/`factor`/`ecdlog`，逼玩家手搜。
- 階段 A 王關標 `unlockTool:'dlog'`；過關後該卡進 `tools`，於階段 B 與後續章節可用。
- 教學弧：**暴力可行（小）→ 演算法勝暴力（中，BSGS 拆「建表／對撞」兩卡）→ 連演算法都跪（大）＝安全性的體感證明**。

對應三個破解王：RSA→`factor`、DLP→`dlog`、ECDLP→`ecdlog`，全部走同一兩段式模板。

## 6. 計分 · par · 提示

- **par** = 該關標準解 `path.length`（資料已存在）。`steps ≤ par` → ★完美；解開但較長 → 仍過關。
- **提示**三級，揭示即記 `hintsUsed`，不擋過關只記錄誰需要扶：
  - Lv1：圈出「這關該用哪幾張卡」（縮小搜索）
  - Lv2：給公式骨架 `▢^▢ mod ▢`（標角色、留空）
  - Lv3：完整解（等於投降看答案，仍可過、但成績標記）
- **不做的事**：不因步數多而失敗、不因用提示而鎖關。懲罰一律改成「記錄」。

## 10. why 閘題庫（D7 落地）

每章王關**達標後、蓋 Q.E.D. 前**彈出一題概念挑戰，瞄準該章最常見的誤解。機制：選對才過王關蓋章；選錯不扣關、不重玩，只能重選（呼應零懲罰）。可內建成 `goal` 的附加條件，或獨立 `whyGate` 欄位。

> 設計原則：誘餌選項刻意放**真實流傳的錯誤心智模型**，選錯本身就是教學。正解常常是「回報無解／反直覺的那個」。

### Ch0 質料準備 — 逆元存在條件
> **題幹**：下列哪個數在 `mod 12` 下**沒有**乘法逆元？
> - (A) 5　(B) 7　**(C) 8 ✓**
> - 正解 C：gcd(8,12)=4≠1，無逆元；5、7 與 12 互質故有逆元。
> - **教學點**：`a` 在 mod m 有逆元 ⟺ gcd(a,m)=1。這條是 RSA、仿射、CRT 全部的地基。

### Ch1 黑化 — 古典金鑰的合法性
> **題幹**：仿射密碼 `c=(a·m+b) mod 26`，下列哪個乘數 `a` 會讓密文**無法唯一解密**？
> - (A) a=3　(B) a=7　**(C) a=13 ✓**
> - 正解 C：gcd(13,26)=13≠1，映射非單射，多個明文撞同一密文 → 收方解不開。
> - **教學點**：金鑰乘數須與字母表大小互質（接 Ch0 逆元條件）。順帶體會「古典密碼金鑰空間極小」＝為何不安全。

### Ch2 白化 — e 與 φ 必須互質（impossible 謂詞示範）
> **題幹**：RSA 取 p=11, q=13（故 φ=120），公開指數 `e=10`。能求出私鑰 d 嗎？
> - (A) 能，d 是某個介於 0–120 的數
> - **(B) 不能：gcd(10,120)=10≠1，e 在 mod φ 下無逆元 → 回報無解 ✓**
> - (C) 不能：因為 e 必須是質數
> - 正解 B（正解＝按「回報無解」鈕，引擎驗證確實無 d）。
> - **教學點**：d=e⁻¹ mod φ 存在 ⟺ gcd(e,φ)=1。誘餌 C 殺掉「e 必須是質數」這個常見誤解——65537 是質數只是工程上方便，互質才是真條件。

### Ch3 黃化 — 暴力搜索的量級（已於 §8 示範）
> **題幹**：把 DLP 的質數 p 從 23（5-bit）換成 2048-bit，用你剛玩的「逐一試 x」暴力法，最壞要試幾次？
> - (A) 約 2048 次　(B) 約 2048² 次　**(C) 約 2²⁰⁴⁸ 次 ✓**
> - 正解 C：候選約 p 個，p≈2²⁰⁴⁸；步數隨位元數**指數**成長。
> - **教學點**：玩家剛親手搜過小參數，這題把「步數計數器」外推到真實參數 → 安全性的體感證明。承接階段 A/B 的步數對比。

### Ch4 紅化 — 為何 ECC 金鑰能更短
> **題幹**：256-bit 的橢圓曲線金鑰，安全強度大約相當於幾 bit 的 RSA？
> - (A) 256　(B) 512　**(C) 3072 ✓**
> - 正解 C：256-bit ECC ≈ 3072-bit RSA。
> - **教學點**：ECDLP 沒有 RSA／DLP 那種次指數攻擊（數體篩、index-calculus），最佳已知攻擊仍是 √n 級 → 同等安全所需金鑰小一個量級。這是 ECC 在實務勝出的根本原因。

### Ch5 封印 — 兩題（簽章為何要雜湊、生日界）
> **題幹 5-甲（存在性偽造）**：textbook RSA 簽章不雜湊、直接讓 sig 滿足 `m=sigᵉ mod n`。攻擊者**沒有**私鑰，能偽造一組通過驗證的 (m, sig) 嗎？
> - (A) 不能，沒私鑰簽不出任何東西
> - **(B) 能：隨機挑一個 sig，令 m=sigᵉ mod n，這對 (m,sig) 必過驗證 ✓**
> - (C) 能，但要先分解 n
> - 正解 B。**教學點**：必須簽 `H(m)` 而非 m——否則攻擊者可從任意 sig 反推出一個合法 m，造出有效簽章（存在性偽造）。直接呼應 5.3 偽造關。
>
> **題幹 5-乙（生日界）**：玩具雜湊輸出 n bits（2ⁿ 種值），平均要試幾次才找到一組碰撞 `H(x)=H(y)`？
> - (A) 2ⁿ　**(B) 2^(n/2) ✓**　(C) n
> - 正解 B。**教學點**：生日悖論 → 抗碰撞強度只有輸出長度的一半，所以要 256-bit 雜湊才有 128-bit 抗碰撞。連到 5.4 生日碰撞關。

## 11. 題目分類體系 · 類別驅動重編（v2 lineup）

> v1 的 29 關只是草稿，分佈失衡（≈70% 計算、推導幾乎掛零、攻擊只在王關且用萬能卡造假）。
> v2 改成**用四類認知階梯當編排骨架**：每關掛一個主類，每章被要求走完階梯，不准整章全計算。

### 11.1 四類定義（每關只掛一個主類）

| 類別 | 學生在做什麼 | Bloom | 謂詞(D3) | 設計手法 |
|---|---|---|---|---|
| **計算** Compute | 運算與公式已給，照著求出值 | Apply | `equals` | 熱身、建立操作詞彙；數量壓到最少 |
| **推導** Derive | 自己選運算、組方法鏈、找路徑 | Understand→Analyze | `equals`/`tokensEqual` | 拔公式（去公式化 story）、手牌給誘餌卡 |
| **攻擊** Attack | 反求單向函數、偽造、找碰撞 | Analyze→Evaluate | `inverts`/`forges` | v2 兩段式真窮舉，**不發萬能卡** |
| **證明** Prove | 建一條引擎可逐步驗證的鏈，確立某*命題* | Evaluate | `forges`/`impossible`＋PDF 轉錄 | 章末王；命題＝「一定還原／恆相等／可偽造」＋why 閘 |

> 「證明」＝確立命題（合法簽章、RSA 必還原、K_A≡K_B），不是「算出某個數」。那條可驗證鏈本身就是證明，呼應 Q.E.D.＋PDF 轉錄品牌。

### 11.2 編排原則

1. **單一主類**：一關只一類。歸不進單一類 → 目的不清，該拆。
2. **章為階梯**：每章至少含「推導」＋（「攻擊」或「證明」）；禁止整章全計算。
3. **計算當 onramp**：只建操作熟練度，每章壓 1–2 關。
4. **王＝證明關**：建可驗證鏈＋why 閘收尾。
5. **攻擊用兩段式**：先親手窮舉（小參數）→解鎖工具（大參數），步數＝難度量尺（§5）。
